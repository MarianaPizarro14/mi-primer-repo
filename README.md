## 🌿 Uso de ramas en GitHub

Las ramas en GitHub permiten trabajar en nuevas funciones o cambios sin afectar el proyecto principal. Sirven para organizar mejor el trabajo en equipo y evitar conflictos mientras cada integrante desarrolla su parte.

En nuestro proyecto **"Uy, era para hoy"**, utilizaremos las ramas para separar tareas y funcionalidades. Por ejemplo:

- Una rama para el diseño de la interfaz en Vue.js.
- Otra rama para las alertas y notificaciones.
- Otra para el calendario inteligente.

Cuando una funcionalidad esté terminada y probada, se combinará con la rama principal (`main`) mediante un merge.

De esta manera, el equipo podrá trabajar al mismo tiempo de forma más ordenada y segura.
